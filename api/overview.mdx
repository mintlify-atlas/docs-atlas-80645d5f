## Introduction

The Click Creators Scraper Client backend API provides REST endpoints for managing Instagram follower scraping, profile ingestion, campaign creation, VA distribution, and Airtable synchronization.

This Flask-based API integrates with:
- **Apify** - Instagram scraping service
- **Supabase** - PostgreSQL database
- **Airtable** - VA workspace management
- **Gender API** - Name-based gender detection

## Base URL

The API base URL is configurable via environment variables:

<CodeGroup>
```bash Development
http://localhost:5001
```

```bash Production
https://api.yourplatform.com
```
</CodeGroup>

Set the base URL in your frontend environment:

```bash .env.local
NEXT_PUBLIC_API_URL=http://localhost:5001
```

## Authentication

### Multi-Tenant Base ID

All API requests require a base identifier in the request headers:

```bash
X-Base-Id: your-airtable-base-id
```

This header enables multi-tenant support, isolating data between different Airtable workspaces.

### Using the API Client

The frontend provides helper functions in `lib/api.ts`:

```typescript
import { apiPost, apiGet } from '@/lib/api'
import { useBase } from '@/contexts/base-context'

const { baseId } = useBase()

// POST request with auto base_id injection
const response = await apiPost('/api/scrape-followers', baseId, {
  accounts: ['username1', 'username2'],
  targetGender: 'male'
})

// GET request
const data = await apiGet('/api/status', baseId)
```

## Request Format

All POST requests should include:

<ParamField header="X-Base-Id" type="string" required>

</ParamField>

<ParamField header="Content-Type" type="string" default="application/json">
  Request content type
</ParamField>

## Response Format

All API responses follow a consistent structure:

### Success Response

```json
{
  "success": true,
  "data": {
    // Endpoint-specific data
  }
}
```

### Error Response

```json
{
  "success": false,
  "error": "Error message",
  "details": {
    // Additional error context
  }
}
```

<ResponseField name="200" type="OK">

  Server error occurred

## Rate Limiting

The API implements rate limiting for external service calls:

- **Apify**: Respects actor run quotas
- **Airtable**: Built-in delays between batch uploads (5 requests/second)
- **Gender API**: Batch processing with configurable limits

## Environment Variables

Backend API configuration:

```bash .env
# Supabase
SUPABASE_URL=https://your-project.supabase.co
SUPABASE_KEY=your-service-role-key

# Apify (Instagram Scraper)
APIFY_API_KEY=your-apify-key
APIFY_ACTOR_ID=your-actor-id

# Airtable
AIRTABLE_API_KEY=your-airtable-key
AIRTABLE_BASE_ID=your-base-id

# Gender Detection API (optional)
GENDER_API_KEY=your-gender-api-key
```

## Error Handling

Common error scenarios:

### Missing Base ID

```json
{
  "success": false,
  "error": "X-Base-Id header is required"
}
```

### Invalid Parameters

```json
{
  "success": false,
  "error": "Invalid request parameters",
  "details": {
    "accounts": "Must be a non-empty array"
  }
}
```

### External Service Error

```json
{
  "success": false,
  "error": "Apify scraping failed",
  "details": {
    "message": "Actor run timeout",
    "actorId": "..."
  }
}
```

## Support

For API issues:
1. Check environment variables are set correctly
2. Verify external service credentials (Apify, Airtable)
3. Review backend logs for detailed error messages
4. Test endpoints with curl or Postman
5. Ensure database schema is up to date
