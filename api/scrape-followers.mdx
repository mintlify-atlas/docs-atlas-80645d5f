## Description

Scrapes Instagram followers from specified source accounts using the Apify Instagram scraper. Applies gender-based filtering using name detection to identify male profiles.

This endpoint:
1. Validates source account usernames
2. Initiates Apify actor runs for each account
3. Extracts follower profile data
4. Applies gender detection (male/female/unknown)
5. Filters results based on target gender
6. Returns filtered profile list

<Info>
  This endpoint does **not** save profiles to the database. Use `/api/ingest` to save the returned profiles.
</Info>

## Request Body

<ParamField body="accounts" type="string[]" required>
  Array of Instagram usernames or profile URLs to scrape followers from.
  
  Examples:
  - `["username1", "username2"]`
  - `["https://instagram.com/username1"]`
  
  **Validation:**
  - Must be a non-empty array
  - Each account must be a valid Instagram username or URL
  - Public accounts only (private accounts will return no followers)
</ParamField>

<ParamField body="targetGender" type="string" default="male">
  Gender filter to apply during scraping.
  
  **Options:**
  - `"male"` - Return only male profiles
  - `"female"` - Return only female profiles
  - `"all"` - Return all profiles (no filtering)
  
  **Default:** `"male"`
</ParamField>

## Workflow Integration

This endpoint is typically used in conjunction with `/api/ingest`:

```typescript
// Step 1: Scrape followers
const scrapeResponse = await apiPost('/api/scrape-followers', baseId, {
  accounts: sourceAccounts,
  targetGender: 'male'
})

if (scrapeResponse.success) {
  // Step 2: Ingest profiles to database
  const ingestResponse = await apiPost('/api/ingest', baseId, {
    profiles: scrapeResponse.data.accounts
  })
}
```

## Performance Considerations

<Warning>
  Scraping time varies based on:
  - Number of source accounts
  - Follower counts per account
  - Apify actor performance
  - Network conditions
  
  Expect 30-120 seconds for typical workloads.
</Warning>

**Optimization Tips:**
- Batch source accounts (2-5 accounts per request)
- Monitor Apify run quotas
- Use public accounts only (private accounts return no data)
- Consider running during off-peak hours

## Error Handling

Common error scenarios:

### Invalid Instagram Accounts

**Cause:** Account doesn't exist or is private

**Solution:** Verify accounts are public and exist on Instagram

### Apify Quota Exceeded

**Cause:** Monthly Apify run limit reached

**Solution:** Upgrade Apify plan or wait for quota reset

### Timeout

**Cause:** Scraping took longer than 5 minutes

**Solution:** Reduce number of source accounts or retry
