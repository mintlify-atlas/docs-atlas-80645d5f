## Overview

The `global_usernames` table serves as the **deduplicated username pool** for all scraped Instagram profiles. It tracks which usernames have been used in campaigns and provides the available username count for daily assignments.

<Info>
This table ensures that each username is only assigned once across all campaigns, preventing duplicate outreach to the same users.
</Info>

## Schema Definition

```sql
CREATE TABLE global_usernames (
  id TEXT PRIMARY KEY,  -- Instagram profile ID
  username TEXT NOT NULL,
  full_name TEXT,
  used BOOLEAN DEFAULT FALSE,
  used_at TIMESTAMPTZ,
  job_id UUID REFERENCES scraping_jobs(job_id),
  base_id TEXT NOT NULL,  -- Multi-tenant isolation key
  created_at TIMESTAMPTZ DEFAULT NOW()
);
```

## Indexes and Constraints

```sql
-- Primary key index (profile ID)
CREATE INDEX idx_global_usernames_pkey ON global_usernames(id);

-- Username lookup
CREATE INDEX idx_global_usernames_username ON global_usernames(username);

-- Used status filtering (critical for availability queries)
CREATE INDEX idx_global_usernames_used ON global_usernames(used);

-- Multi-tenant isolation
CREATE INDEX idx_global_usernames_base_id ON global_usernames(base_id);

-- Job association
CREATE INDEX idx_global_usernames_job_id ON global_usernames(job_id);

-- Composite index for available usernames query
CREATE INDEX idx_global_usernames_base_used ON global_usernames(base_id, used);

-- Foreign key constraint
ALTER TABLE global_usernames
  ADD CONSTRAINT fk_global_usernames_job
  FOREIGN KEY (job_id)
  REFERENCES scraping_jobs(job_id)
  ON DELETE SET NULL;
```

## Common Queries

### Get Available Username Count

```typescript
import { supabase } from '@/lib/supabase'
import { getActiveBaseId } from '@/lib/scraping-jobs'

const baseId = await getActiveBaseId()
const { count } = await supabase
  .from('global_usernames')
  .select('*', { count: 'exact', head: true })
  .eq('base_id', baseId)
  .eq('used', false)

console.log(`Available usernames: ${count}`)
```

### Select Unused Usernames for Campaign

```typescript
const DAILY_TARGET = 14400

const { data: usernames, error } = await supabase
  .from('global_usernames')
  .select('id, username, full_name')
  .eq('base_id', baseId)
  .eq('used', false)
  .limit(DAILY_TARGET)

if (usernames && usernames.length >= DAILY_TARGET) {
  console.log('Sufficient usernames available')
} else {
  console.log('Insufficient usernames, need to scrape more')
}
```

### Mark Usernames as Used

```typescript
const usernameIds = ['id1', 'id2', 'id3'] // Profile IDs

const { error } = await supabase
  .from('global_usernames')
  .update({
    used: true,
    used_at: new Date().toISOString()
  })
  .in('id', usernameIds)
  .eq('base_id', baseId) // Multi-tenant safety
```

### Add Usernames from Scraping

```typescript
import { addUsernamesToJob } from '@/lib/scraping-jobs'
```

## Usage Tracking

### Campaign Assignment Flow

```typescript
// 1. Select unused usernames
const { data: selectedUsernames } = await supabase
  .from('global_usernames')
  .select('*')
  .eq('base_id', baseId)
  .eq('used', false)
  .limit(14400)

// 2. Create campaign
const { data: campaign } = await supabase
  .from('campaigns')
  .insert({
    campaign_date: new Date().toISOString().split('T')[0],
    total_assigned: selectedUsernames.length,
    status: 'pending',
    base_id: baseId
  })
  .select()
  .single()

// 3. Mark usernames as used
const ids = selectedUsernames.map(u => u.id)
await supabase
  .from('global_usernames')
  .update({ used: true, used_at: new Date().toISOString() })
  .in('id', ids)
  .eq('base_id', baseId)

// 4. Create assignments (see daily_assignments table)
```

## Deduplication Strategy

### Two-Table Architecture

The system uses a two-table strategy for deduplication:

1. **raw_scraped_profiles** (not shown in schema) - Stores all scraped data
2. **global_usernames** - Deduplicated pool based on Instagram profile ID

```typescript
// Scraping ingest process
const scrapedProfiles = [...] // From Apify scraper

// Insert into raw_scraped_profiles (keeps all data)
await supabase.from('raw_scraped_profiles').insert(scrapedProfiles)

// Deduplicate into global_usernames
for (const profile of scrapedProfiles) {
  await supabase
    .from('global_usernames')
    .upsert({
      id: profile.id,
      username: profile.username,
      full_name: profile.full_name,
      job_id: jobId,
      base_id: baseId
    }, {
      onConflict: 'id' // Skip if profile ID already exists
    })
}
```

## Multi-Tenant Isolation

### RLS Policy

```sql
CREATE POLICY "Isolate global_usernames by base_id"
  ON global_usernames
  FOR ALL
  USING (
    base_id = current_setting('request.headers')::json->>'x-base-id'
  );

ALTER TABLE global_usernames ENABLE ROW LEVEL SECURITY;
```

## Best Practices

<Check>**Use profile ID as primary key** - Ensures global uniqueness across all scraping operations</Check>

<Check>**Check availability before campaign creation** - Verify sufficient unused usernames exist</Check>

<Check>**Mark as used atomically** - Update used flag in same transaction as assignment creation</Check>

<Warning>**Never reset 'used' flag** - Once used, usernames should remain marked to prevent duplicates</Warning>

<Warning>**Always filter by base_id** - Critical for multi-tenant data isolation</Warning>
