## Overview

The Airtable Sync system is the final phase of campaign creation, pushing distributed profiles from Supabase to Airtable tables where virtual assistants can access and process them through the Airtable interface.

```typescript
const validateAirtableLink = (link: string): boolean => {
  // Basic validation for Airtable URL pattern
  const airtablePattern = /^https?:\/\/(airtable\.com|www\.airtable\.com)\/.+/i;
  return airtablePattern.test(link);
};

const handleSubmit = () => {
  const trimmedLink = airtableLink.trim();
  
  if (!trimmedLink) {
    setError('Airtable link is required');
    return;
  }

  if (!validateAirtableLink(trimmedLink)) {
    setError('Please enter a valid Airtable URL');
    return;
  }

  onSubmit(trimmedLink);
};
```

## Airtable Table Structure

### Individual VA Table Schema

Each of the 80 VA tables contains:

```typescript
interface AirtableRecord {
  fields: {
    position: number;           // 1-180 (sequential order)
    username: string;           // Instagram/TikTok/etc username
    full_name: string | null;  // Display name
    profile_id: string;        // Unique identifier
    status: string;            // 'pending' | 'followed' | 'unfollow' | 'completed'
    assigned_at: string;       // ISO timestamp
    campaign_id: string;       // Campaign reference
  }
}
```

### Table Naming Convention

```
Table 1   → VA Assistant #1
Table 2   → VA Assistant #2
...
Table 80  → VA Assistant #80
```

Each table is numbered sequentially and contains exactly `profiles_per_table` records (default: 180).

## Sync Implementation Details

### Rate Limit Handling

Airtable API has rate limits that must be respected:

```
Rate Limits:
- 5 requests per second per base
- Built-in delays between batch uploads
```

Source: README.md:684

### Batch Upload Strategy

```typescript
// Conceptual backend implementation
const syncToAirtable = async (campaignId: string, baseId: string) => {
  // Get all assignments for this campaign
  const assignments = await getAssignmentsByCampaign(campaignId, baseId);
  
  // Group by VA table number
  const groupedByVA = groupBy(assignments, 'va_table_number');
  
  // Sync each VA table sequentially
  for (const [tableNumber, records] of Object.entries(groupedByVA)) {
    await syncTableToAirtable(
      airtableBaseId,
      `Table ${tableNumber}`,
      records
    );
    
    // Rate limit delay
    await sleep(200);  // 5 requests/second = 200ms delay
  }
};
```

## Error Handling

### Common Sync Errors

<AccordionGroup>
  <Accordion title="Invalid Airtable URL">
    **Error:** `Please enter a valid Airtable URL`
    
    **Cause:** URL doesn't match Airtable pattern
    
    **Solution:** Ensure URL is copied directly from Airtable browser tab:
    ```
    ✅ https://airtable.com/appXXXXXXXXXXXXXX/...
    ❌ airtable.com/...
    ❌ https://www.notion.com/...
    ```
    
    Source: `components/airtable-link-dialog.tsx:49-52`
  </Accordion>
  
  <Accordion title="Airtable API Authentication Failed">
    **Error:** `Airtable sync failed: Authentication error`
    
    **Cause:** Invalid or expired Airtable API key
    
    **Solution:** Verify `AIRTABLE_API_KEY` in backend `.env`:
    ```bash
    AIRTABLE_API_KEY=your-airtable-key
    ```
    
    Source: README.md:514-516
  </Accordion>
  
  <Accordion title="Rate Limit Exceeded">
    **Error:** `Airtable sync failed: Rate limit exceeded`
    
    **Cause:** Too many requests to Airtable API
    
    **Solution:** Backend should implement exponential backoff:
    ```typescript
    const retryWithBackoff = async (fn, maxRetries = 3) => {
      for (let i = 0; i < maxRetries; i++) {
        try {
          return await fn();
        } catch (error) {
          if (error.statusCode === 429 && i < maxRetries - 1) {
            await sleep(Math.pow(2, i) * 1000);  // 1s, 2s, 4s
            continue;
          }
          throw error;
        }
      }
    };
    ```
    
    Source: README.md:623-625
  </Accordion>
  
  <Accordion title="Base Not Found">
    **Error:** `Airtable sync failed: Base not found`
    
    **Cause:** Airtable base was deleted or URL is incorrect
    
    **Solution:** 
    1. Verify base still exists in Airtable workspace
    2. Update job with new Airtable base URL
    3. Ensure workspace permissions are correct
  </Accordion>
</AccordionGroup>

### Error Recovery

```typescript
    variant: "destructive"
```

## Best Practices

<AccordionGroup>
  <Accordion title="Always create base before job creation">
    Create and name the Airtable base before starting job creation to avoid workflow interruption:
    
    1. Navigate to workspace
    2. Create base with correct naming
    3. Copy URL immediately
    4. Have URL ready when creating job
  </Accordion>
  
  <Accordion title="Verify base permissions">
    Ensure the Airtable API key has proper permissions:
    
    - ✅ Create records
    - ✅ Read records  
    - ✅ Update records
    - ✅ Delete records (for cleanup)
    
    Test with a small campaign first before large-scale operations.
  </Accordion>
  
  <Accordion title="Monitor rate limits">
    Keep track of API usage to avoid rate limit errors:
    
    ```typescript
    // Check Airtable API usage
    const estimatedRequests = Math.ceil(14400 / 10);  // 10 records per request
    const estimatedTime = estimatedRequests * 0.2;    // 200ms per request
    
    console.log(`Estimated sync time: ${estimatedTime}s`);
    // ~288 seconds = 4.8 minutes for 14,400 records
    ```
  </Accordion>
  
  <Accordion title="Use meaningful base names">
    Follow consistent naming convention for easy identification:
    
    ```
    ✅ Good:
    - "John Doe's Instagram Campaign"
    - "Nike's TikTok Campaign"
    
    ❌ Bad:
    - "Test"
    - "Campaign 1"
    - "Untitled Base"
    ```
    
    This helps VAs quickly identify which campaign they're working on.
  </Accordion>
  
  <Accordion title="Handle sync failures gracefully">
    Always provide clear error messages and recovery options:
    
    ```typescript
    if (!syncResult.success) {
      // Log detailed error
      console.error('Airtable sync failed:', syncResult.error);
      
      // Show user-friendly message
      toast({
        title: "Sync Failed",
        description: "Please check Airtable connection and try again.",
        action: (
          <Button onClick={() => retrySync(campaignId)}>Retry</Button>
        )
      });
    }
    ```
  </Accordion>
</AccordionGroup>
