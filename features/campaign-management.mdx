## Overview

Campaigns represent daily batches of profiles distributed to virtual assistants (VAs). The campaign system automatically selects unused profiles, distributes them across VA tables, and syncs to Airtable—all in one streamlined workflow.

## Campaign Data Structure

### Database Schema

```sql
CREATE TABLE campaigns (

  campaign_date DATE NOT NULL,
  total_assigned INTEGER NOT NULL,
  status BOOLEAN,  -- true = success, false = failed
```

### Campaign Filtering by Job

Campaigns are automatically filtered by active job's `base_id`:

```typescript
useEffect(() => {
  if (!isLoadingBase && baseId) {
    fetchCampaigns();
  }
}, [baseId, isLoadingBase]);

const fetchCampaigns = async () => {
  const supabaseWithContext = createSupabaseClientWithContext(baseId);
  
  const { data, error } = await supabaseWithContext
    .from('campaigns')
    .select('campaign_id, campaign_date, total_assigned, status, created_at')
    .eq('base_id', baseId)  // Filter by active job's base_id
    .order('campaign_date', { ascending: false });
  
  setCampaigns(data || []);
}
```

## Profile Selection Logic

### Unused Username Selection

The backend selects profiles marked as `used = false`:

```sql
-- Query executed by /api/daily-selection
SELECT id, username, full_name
FROM global_usernames
WHERE base_id = ? 
  AND used = false
ORDER BY created_at DESC
LIMIT ?  -- profiles_per_table × 80
```

### Marking Profiles as Used

Once selected, profiles are marked to prevent duplicate assignments:

```sql
-- Update query in /api/daily-selection
UPDATE global_usernames
SET used = true, used_at = NOW()
WHERE id IN (selected_profile_ids)
  AND base_id = ?;
```

## Best Practices

<AccordionGroup>
  <Accordion title="Verify profile availability before campaigns">
    Always check `availableUsernames` count before creating campaigns:
    
    ```typescript
    const stats = await getJobStatisticsByBaseId(baseId);
    
    if (stats.availableUsernames < 14400) {
      // Show warning, prevent campaign creation
      return;
    }
    ```
  </Accordion>
  
  <Accordion title="Monitor progress with global context">
    Use `useAssignmentProgress()` to track campaign state across components:
    
    ```typescript
    const { isAssigning, progress } = useAssignmentProgress();
    
    // Disable UI during assignment
    <Button disabled={isAssigning}>Create Campaign</Button>
    ```
  </Accordion>
  
  <Accordion title="Handle all three phases sequentially">
    Never skip phases or run in parallel—each phase depends on the previous:
    
    ```typescript
    // ✅ Correct: Sequential execution
    const campaign = await createCampaign();
    const distributed = await distribute(campaign.id);
    const synced = await syncToAirtable(campaign.id);
    
    // ❌ Wrong: Parallel execution will fail
    await Promise.all([
      createCampaign(),
      distribute(),  // Missing campaign_id!
      syncToAirtable()  // Missing campaign_id!
    ]);
    ```
  </Accordion>
  
  <Accordion title="Reset state when switching jobs">
    Clear campaign data when `baseId` changes:
    
    ```typescript
    useResetOnChange(baseId, () => {
      setCampaigns([]);
      setIsLoading(true);
      setError(null);
    ```
