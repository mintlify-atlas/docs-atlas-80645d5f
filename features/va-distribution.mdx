## Overview

The VA Distribution system automatically divides selected profiles across 80 virtual assistant (VA) tables, ensuring even distribution and tracking each profile's position for ordered workflow execution.

## Distribution Architecture

### Daily Assignments Table

Profile assignments are stored with complete tracking metadata:

```sql
CREATE TABLE daily_assignments (
  assignment_id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  campaign_id UUID REFERENCES campaigns(campaign_id),
  va_table_number INTEGER NOT NULL,  -- 1-80
  position INTEGER NOT NULL,          -- Sequential position within VA table
  id TEXT NOT NULL,                   -- Profile ID from global_usernames
  username TEXT NOT NULL,
  full_name TEXT,
  base_id TEXT NOT NULL,              -- Multi-tenant isolation
  assigned_at TIMESTAMP DEFAULT NOW(),
  status TEXT DEFAULT 'pending',      -- 'pending' | 'followed' | 'unfollow' | 'completed'
  updated_at TIMESTAMP
);
```

## Distribution Workflow

### Distribution Algorithm

<Steps>
  <Step title="Calculate Distribution">
    Determine how many profiles each VA receives:
    
    ```typescript
    const profilesPerTable = 180;  // Configurable
    const numVAs = 80;
    const totalProfiles = profilesPerTable * numVAs;  // 14,400
    ```
    
    Source: `components/payments-table.tsx:112`
  </Step>
  
  <Step title="Round-Robin Assignment">
    Distribute profiles evenly across VA tables:
    
    ```typescript
    // Backend distribution logic (conceptual)
    const assignments = [];
    
    for (let i = 0; i < selectedProfiles.length; i++) {
      const profile = selectedProfiles[i];
      const vaTableNumber = (i % 80) + 1;  // Tables 1-80
      const position = Math.floor(i / 80) + 1;  // Position within table
      
      assignments.push({
        campaign_id: campaignId,
        id: profile.id,
        username: profile.username,
        full_name: profile.full_name,
        va_table_number: vaTableNumber,
        position: position,
        base_id: baseId,
        status: 'pending'
      });
    }
    ```
    
    **Example:** 14,400 profiles → 180 profiles per VA table
    - Profile 1 → VA Table 1, Position 1
    - Profile 2 → VA Table 2, Position 1
    - ...
    - Profile 80 → VA Table 80, Position 1
    - Profile 81 → VA Table 1, Position 2
    - Profile 82 → VA Table 2, Position 2
  </Step>
  
  <Step title="Batch Insert">
    Insert all assignments in a single database transaction:
    
    ```typescript
    export async function createJobAssignments(
      jobId: string,
      campaignId: string,
      baseId: string,
      assignments: Array<{
        id: string;
        username: string;
        full_name?: string;
        va_table_number: number;
        position: number;
      }>
    ): Promise<boolean> {
      const { error } = await supabase
        .from('daily_assignments')
        .insert(
          assignments.map(a => ({
            campaign_id: campaignId,
            id: a.id,
            username: a.username,
            full_name: a.full_name || null,
            va_table_number: a.va_table_number,
            position: a.position,
            job_id: jobId,
            status: 'pending',
            base_id: baseId
          }))
        );
      
      return !error;
    }
    ```
    
    Source: `lib/scraping-jobs.ts:510-555`
  </Step>
</Steps>


## Assignment Status Tracking

### Status Lifecycle

<Steps>
  <Step title="pending">
    Initial state when profile is assigned to VA table
    
    - Profile created in `daily_assignments`
    - Synced to Airtable
    - Awaiting VA action
  </Step>
  
  <Step title="followed">
    VA has followed the Instagram account
    
    - Status updated by VA or automation
    - Profile still active in workflow
  </Step>
  
  <Step title="unfollow">
    VA has unfollowed the account (after engagement period)
    
    - Typically 7 days after initial follow
    - Profile nearing completion
  </Step>
  
  <Step title="completed">
    Workflow finished for this profile
    
    - All actions completed
    - Profile can be archived or deleted
  </Step>
</Steps>

### Status Update Query

```typescript
// Update assignment status
await supabase

  .update({ 
    status: 'followed',
    updated_at: new Date().toISOString()
  })
  .eq('assignment_id', assignmentId)
  .eq('base_id', baseId);
```

## Best Practices

<AccordionGroup>
  <Accordion title="Verify even distribution">
    After distribution, verify each VA table received equal profiles:
    
    ```typescript
    const verifyDistribution = async (campaignId: string, baseId: string) => {
      const counts = await Promise.all(
        Array.from({ length: 80 }, async (_, i) => {
          const vaTable = i + 1;
          const { count } = await supabase
            .from('daily_assignments')
            .select('*', { count: 'exact', head: true })
            .eq('base_id', baseId)
            .eq('campaign_id', campaignId)
            .eq('va_table_number', vaTable);
          return count;
        })
      );
      
      // All counts should be equal
      const allEqual = counts.every(c => c === counts[0]);
      console.log('Distribution even:', allEqual);
    };
    ```
  </Accordion>
  
  <Accordion title="Maintain position ordering">
    Always query with position ordering for consistent workflow:
    
    ```typescript
    const { data } = await supabase
      .from('daily_assignments')
      .select('*')
      .eq('va_table_number', tableNumber)
      .order('position', { ascending: true });  // Important!
    ```
  </Accordion>
  
  <Accordion title="Update status atomically">
    Use transactions or row-level locks when updating assignment status:
    
    ```typescript
    // Good: Single atomic update
    await supabase
      .from('daily_assignments')
      .update({ status: 'followed', updated_at: NOW() })
      .eq('assignment_id', id);
    
    // Avoid: Read-then-write (race condition)
    const assignment = await getAssignment(id);
    assignment.status = 'followed';
    await updateAssignment(assignment);
    ```
  </Accordion>
  
  <Accordion title="Include base_id in all queries">
    Never query assignments without base_id filter:
    
    ```typescript
    // ✅ Correct: Includes base_id
    await supabase
      .from('daily_assignments')
      .select('*')
      .eq('base_id', baseId)
      .eq('va_table_number', 5);
    
    // ❌ Wrong: Missing base_id (data leak!)
    await supabase
      .from('daily_assignments')
      .select('*')
      .eq('va_table_number', 5);  // Returns all jobs' data!
    ```
  </Accordion>
</AccordionGroup>
